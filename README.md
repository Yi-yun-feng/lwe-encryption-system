# 基于LWE的轻量化可搜索加密系统

**Lightweight Searchable Encryption System Based on LWE**

---

## 📌 项目简介

本项目设计并实现了一个基于学习带误差问题（Learning With Errors, LWE）的轻量化可搜索加密系统。系统支持在加密数据上执行关键词搜索，在不暴露明文信息的前提下，实现安全高效的数据检索。

项目结合 Web 前后端架构，实现了文件加密上传、密文存储、关键词搜索以及结果解密展示等功能，适用于云存储、隐私计算等场景。

---

## 🌍 背景与意义

随着云计算和数据外包的发展，用户越来越依赖第三方服务器存储数据。然而，直接上传明文数据存在严重的隐私泄露风险。

可搜索加密（Searchable Encryption, SE）技术允许用户在密文上进行搜索操作，在保证数据隐私的同时实现数据可用性。

传统可搜索加密方案大多基于经典密码学构造，难以抵抗量子攻击。LWE问题被认为是后量子密码的重要基础，其安全性基于格困难问题。

本项目基于LWE构建轻量级可搜索加密系统，在保证安全性的同时，优化系统复杂度与实际可部署性。

---

## 🧱 系统架构

系统采用分层架构设计：

* **前端层**：用户交互界面（文件上传、关键词输入、结果展示）
* **后端层**：请求处理与逻辑控制（Flask）
* **密码层**：LWE加密、Trapdoor生成与匹配
* **存储层**：密文数据库与关键词索引

### 系统流程

1. 用户上传文件并输入关键词
2. 系统对文件和关键词进行加密
3. 密文与索引存储至服务器
4. 用户输入关键词生成Trapdoor
5. 服务器执行搜索并返回匹配密文
6. 用户解密获得原始数据

## 🔬 技术路线

### 1. 关键词编码

将关键词通过哈希函数映射为向量表示：

keyword → hash → vector

---

### 2. LWE加密

对数据和关键词向量进行加密：

Enc(m) = (A, A·s + e + m)

其中：

* A 为随机矩阵
* s 为秘密向量
* e 为误差项

---

### 3. Trapdoor生成

用户基于私钥生成搜索令牌：

Trapdoor(sk, w) → T_w

---

### 4. 搜索机制

服务器在密文域执行匹配：

* 比较加密关键词与Trapdoor
* 返回匹配结果对应的密文

---

### 5. 轻量化设计

* 使用低维LWE参数
* 简化索引结构
* 采用线性扫描或简单索引
