📌 项目简介
本项目实现了一个基于学习带误差问题（LWE） 的轻量化可搜索加密系统。用户可将文件加密后上传至服务器，并在不泄露明文的前提下，通过关键词检索密文数据。系统结合前后端架构，提供完整的加密上传、密文检索、解密展示流程，并内置性能对比模块，验证后量子安全与轻量化设计的有效性。

🌍 背景与意义
云计算带来数据外包的隐私风险，可搜索加密是解决“加密数据上检索”的关键技术。

传统可搜索加密方案（如RSA、双线性对）在量子计算下不再安全。

LWE是后量子密码的基石，基于格构造的方案能抵抗已知量子攻击。

本项目在保证后量子安全的前提下，通过参数优化与实现调优，实现轻量化部署。

🧱 系统架构
text
前端（HTML/CSS/JS） ↔ Flask后端 ↔ 密文存储（内存/文件）
前端：文件上传、关键词输入、结果展示、性能对比图表

后端：存储密文与安全索引、匹配计算

密码模块：LWE加解密、陷门生成、匹配算法

核心流程：

用户上传文件，输入关键词 → 前端LWE加密文件与关键词 → 后端存储

用户查询关键词 → 前端生成陷门 → 后端匹配 → 返回匹配文档 → 前端解密展示

⚙️ 核心功能
🔐 文件加解密：基于LWE公钥加密

🔍 关键词密文检索：支持精确检索

🗝 陷门生成：用户私钥生成，服务器无法反推关键词

📊 性能对比：与明文检索、RSA-SE、基础LWE-SE对比，动态展示加密/检索耗时、存储开销

🖥️ Web界面：简洁交互，一键演示

🔬 技术路线
关键词编码：SHA-256 → 二进制向量

LWE加密（Regev型）：公钥 (A, b = A·s + e)，密文 (u, v)

可搜索加密：关键词密文 + 陷门 + 内积匹配

轻量化设计：

参数 n=256, q≈2^16, σ=3.2（安全强度≥128-bit）

线性索引，NumPy向量化匹配

前端Web Worker异步加解密

📊 实验摘要
在1000份文档规模下，本方案检索时间约 70ms，准确率 95%，存储开销仅为 1.8KB/关键词。详细对比数据见 docs/experiments.md。

🔒 安全性分析
安全模型：满足IND-CPA（基于LWE假设证明），陷门不可区分。

安全归约：存在攻击则能解LWE问题，因此方案在LWE困难性下安全。

参数安全：经FPLLL评估，抵抗BKZ攻击强度≥128-bit。
